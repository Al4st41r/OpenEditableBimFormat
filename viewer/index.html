Open Editable BIM Format

OEBF

A path-first, human-readable building model format designed for direct editing by people and language models alike. All geometry is stored as plain JSON — paths, profiles, elements, junctions — in a structured directory bundle.

Applications

Viewer

Inspect OEBF bundles and compressed archives in 3D. Includes a sample model you can load immediately.

  • Click Load demo to open the sample model
  • Open any .oebf folder (Chrome/Edge) or .oebfz file (all browsers)
  • Use View profiles to inspect wall and slab cross-sections
Editor

Draw walls and slabs, manage storeys, add reference grids and guides. Saves directly to an .oebf folder or downloads as a .oebfz archive.

  • Open an existing .oebf folder or .oebfz file
  • Select a wall profile and use the Wall tool to draw
  • Save to write changes back to the bundle

How it works

Format Plain JSON, one file per entity

Every wall, path, profile, junction, and storey is a separate JSON file. No binary, no proprietary encoding — open a bundle in any text editor.

Geometry Profile swept along a path

All solid geometry derives from sweeping a cross-section profile along a polyline path. Junctions and slabs are first-class entities, not auto-computed.

Portability Folder or compressed archive

Bundles live as a plain .oebf directory for editing, or are compressed to a single .oebfz file (Zstd) for sharing.

Roadmap

The long-term goal is a full round-trip between IFC and OEBF, with an LLM acting as an intelligent editing layer. Because every entity is a readable JSON file, a language model can add walls, change profiles, or restructure storeys without specialised BIM tooling.

Step 1 Import IFC

Convert an IFC 4x3 file to an OEBF bundle using the included Python tools. Walls, slabs, storeys, and materials are mapped to OEBF entities.

Step 2 Edit with an LLM

Point a language model at the bundle directory. Because files are plain JSON, the model can read, modify, and write entities directly — changing geometry, profiles, or metadata with natural-language instructions.

Step 3 Export to IFC

Convert the modified OEBF bundle back to IFC using the same Python tools. The result opens in Revit, FreeCAD, or any IFC-compatible viewer.

IFC import and export tools are in development. The OEBF format specification is designed to map cleanly to IFC 4x3 entity types including IfcWall, IfcSlab, IfcBuildingStorey, and IfcMaterialLayerSet.
